加载一次spring 数据源配置的坑

发表于2023-01-12|更新于2023-01-14
|字数总计:252|阅读时长:1分钟|阅读量:

在做 批量插入测试事物的时候发现 自动提交始终没有关闭

数据源使用的 druid

spring boot : 2.7.0

问题

不关闭默认的自动提交，这个给创建的不能回退执行flushStatements后的数据

SqlSession sqlSession = sqlSessionFactory
                                .openSession(ExecutorType.BATCH,
		                                                      false);

解决[1]

关闭 数据源的自动提交

spring:
   datasource:
      druid:
         default-auto-commit: false

测试例子

默认设置

  • 将id 第一次修改为 13
  • 将id再次修改为13
  • default-auto-commit: true

id 不可重复

SqlSession sqlSession = sqlSessionFactory.openSession(false);
		TestDao mapper = sqlSession.getMapper(TestDao.class);
		try {
			log.info("--------------开始执行--------------");
			mapper.money(13,
			             "11");
			mapper.money(13,
			             "12");
			log.info("--------------执行结束--------------");
			sqlSession.commit();
		} catch (Exception e) {
			sqlSession.rollback();
		} finally {
			sqlSession.close();
		}
  • 运行结果
    • 第一条成功修改
    • 第二条修改报错SQLIntegrityConstraintViolationException

关闭自动提交

  • 运行结果
    • 两个都没有更新成功

TransactionTemplate不受这个配置影响,事物都会生效

  1. 不影响默认的插入，测试普通的插入仍会提交到数据库中 ↩︎
